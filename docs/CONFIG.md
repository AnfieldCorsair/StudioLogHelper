# StudioLogHelper project config (`.slh.json`)

Файл проекта нужен, чтобы хранить не только сами открытые логи, но и смысловую организацию работы: категории, заметки, связи исходников и экспортов.

Расширение по умолчанию: `.slh.json`.

## Типовой сценарий

1. Создать проект: **Проект и категории → Новый проект**.
2. Открыть 5 исходных файлов без расширения с Google Drive.
3. Создать категорию: `Исходники Google Drive JSON`.
4. Назначить этим файлам категорию и заметку.
5. Экспортировать только ответы модели в отдельную папку.
6. Открыть полученные TXT.
7. Создать категорию: `Только ответы модели по произведению`.
8. Назначить категорию экспортированным TXT.
9. Сохранить проект в `.slh.json`.

Так можно вернуться к рабочему набору позже и сразу видеть, какие файлы являются грязными JSON, какие — очищенными TXT, а какие — ответами модели.

## План развития

Следующие логичные поля:

- `tags`: произвольные теги файла;
- `derived_from`: связь экспортированного TXT с исходным JSON;
- `export_profile`: профиль, которым был создан файл;
- `batch_id`: объединение файлов в одну партию обработки.
